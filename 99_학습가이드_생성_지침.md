# CMOS 학습 가이드 생성 지침

이 문서는 이후 Lecture/Exercise 챕터가 추가될 때, 다른 Codex 인스턴스나 새 세션이 현재 `Study Notes/`의 방식과 같은 품질로 국문 학습 가이드를 계속 만들 수 있도록 정리한 작업 지침이다.

## 최종 목표

학습 가이드는 단순 슬라이드 요약이 아니다. 목표는 사용자가 원본 슬라이드를 다시 열지 않아도 강의의 모든 기술 내용을 이해할 수 있게 만드는 것이다.

반드시 지켜야 할 기준은 다음과 같다.

- 행정 안내, 교수진 소개, 제출 방식처럼 시험/이론 이해에 직접 필요 없는 내용은 제외한다.
- 슬라이드의 기술 내용, 수식, 그래프, 표, 회로, 용어, 명령어는 누락하지 않는다.
- 슬라이드 캡처 이미지는 모든 페이지를 넣을 필요가 없지만, 슬라이드에 있는 내용 자체는 반드시 글로 반영한다.
- 핵심 그림, 회로도, 그래프, 표, 복잡한 수식이 있는 슬라이드는 이미지로 포함하고, 바로 아래에 해설을 붙인다.
- 슬라이드에 짧게만 언급된 이론은 필요한 만큼 보강한다.
- 설명 난이도는 고등학생 수준의 비전공자도 따라올 수 있을 정도로 낮춘다.
- 단, 이상한 비유, 은유, 감성적 표현으로 얼버무리지 않는다. 쉬운 말로 정확하게 설명한다.

## 문서 구조

새 챕터를 추가할 때는 `Study Notes/` 아래에 순번을 붙인 국문 마크다운 파일을 만든다.

```text
Study Notes/
  00_학습가이드_기획.md
  01_CMOS_소자와_스케일링.md
  02_MOSFET_전류_전력_온도.md
  ...
  99_학습가이드_생성_지침.md
```

파일명 규칙:

- 두 자리 순번을 붙인다.
- 주제는 큰 챕터 단위로 잡는다.
- 너무 세세한 슬라이드별 파일을 만들지 않는다.
- 예: `07_아날로그_회로_기초.md`, `08_레이아웃_디자인룰_기생성분.md`

각 문서의 기본 구조는 다음을 따른다.

```markdown
# 07. 챕터 제목

## 이 장의 위치

이 장이 어떤 Lecture/Exercise를 다루는지, 앞 장/뒤 장과 어떻게 연결되는지 설명한다.

## 핵심 질문

이 장을 공부하고 나면 답할 수 있어야 하는 질문을 2-5개 적는다.

## 본문 대주제 1

슬라이드 내용 + 보강 이론 + 이미지 + 수식/그래프 해설을 작성한다.

## 본문 대주제 2

...

## 시험 대비 핵심

중요 개념을 짧은 bullet로 정리한다.

## 포함 범위

- Lecture N: pages x-y
- Exercise M: pages a-b
```

## 전체 작업 순서

새 자료가 들어오면 아래 순서로 진행한다.

1. `Lecture/`, `Exercise/`, 필요하면 `Assignment/`의 새 파일 목록을 확인한다.
2. PDF 페이지 수를 확인한다.
3. 모든 페이지의 텍스트를 추출한다.
4. 페이지별 제목과 핵심 내용을 표로 만든다.
5. 기존 `00_학습가이드_기획.md`의 챕터 분할과 비교해, 새 문서를 만들지 기존 문서에 보강할지 결정한다.
6. 핵심 슬라이드를 이미지로 렌더링해 `Study Notes/assets/slides/`에 저장한다.
7. 슬라이드의 모든 기술 내용을 빠짐없이 문서에 반영한다.
8. 수식, 그래프, 도표, 회로는 의미와 맥락까지 설명한다.
9. 실습 스크립트, netlist, command는 원본을 보존해서 인용하고, 각 줄이 무엇을 뜻하는지 설명한다.
10. 이미지 링크와 문서 누락을 검증한다.

## PDF 텍스트 추출

Windows PowerShell 기준으로, `pdftotext`가 있으면 먼저 사용한다.

```powershell
pdftotext -layout "Lecture\Slides Lecture 10.pdf" -
```

Python과 PyMuPDF가 있으면 페이지별 추출본을 만들 수 있다.

```powershell
$env:PYTHONIOENCODING='utf-8'
@'
import fitz
from pathlib import Path

base = Path(r"C:\Users\pjw98\Documents\TUM\SS26\CMOS_SS26")
out = base / ".omx" / "study_extract" / "text"
out.mkdir(parents=True, exist_ok=True)

files = list((base / "Lecture").glob("*.pdf")) + list((base / "Exercise").glob("*.pdf"))

for pdf in files:
    doc = fitz.open(pdf)
    lines = [f"# {pdf.relative_to(base)}\n"]
    for page_no, page in enumerate(doc, start=1):
        text = page.get_text("text")
        text = "\n".join(line.rstrip() for line in text.splitlines())
        lines.append(f"\n\n## Page {page_no}\n\n{text}\n")
    (out / f"{pdf.stem}.md").write_text("\n".join(lines), encoding="utf-8")

print(f"wrote {len(files)} files to {out}")
'@ | python -
```

추출 후에는 반드시 페이지별로 확인한다. PDF 텍스트 추출은 수식, 첨자, 그리스 문자, 줄바꿈을 자주 깨뜨린다. 따라서 수식은 슬라이드 이미지와 대조해서 다시 써야 한다.

## 슬라이드 이미지 자산 생성

원본 슬라이드 이미지는 아래 폴더에 저장한다.

```text
Study Notes/assets/slides/
```

파일명 규칙:

- `lec02_p12.png`: Lecture 2 page 12
- `ex04_p09.png`: Exercise 4 page 9

렌더링 예시 스크립트:

```powershell
$env:PYTHONIOENCODING='utf-8'
@'
import fitz
from pathlib import Path

base = Path(r"C:\Users\pjw98\Documents\TUM\SS26\CMOS_SS26")
out = base / "Study Notes" / "assets" / "slides"
out.mkdir(parents=True, exist_ok=True)

# 필요한 페이지만 선택한다. 페이지 번호는 1부터 시작한다.
selections = {
    "Lecture/Slides Lecture 10.pdf": range(2, 25),
    "Exercise/Slides Exercise 05.pdf": range(2, 18),
}

for rel, pages in selections.items():
    pdf = base / rel
    doc = fitz.open(pdf)
    kind = "lec" if rel.startswith("Lecture") else "ex"
    number = int("".join(ch for ch in pdf.stem if ch.isdigit())[-2:])

    for page_no in pages:
        if page_no < 1 or page_no > len(doc):
            continue
        page = doc[page_no - 1]
        pix = page.get_pixmap(matrix=fitz.Matrix(1.5, 1.5), alpha=False)
        pix.save(out / f"{kind}{number:02d}_p{page_no:02d}.png")
'@ | python -
```

이미지 포함 기준:

- 회로도, transistor 단면, layout, graph, table, waveform은 우선 포함한다.
- 수식이 복잡하거나 텍스트 추출이 깨지는 슬라이드는 포함한다.
- 단순 제목 슬라이드, 행정 안내, 반복 summary만 있는 슬라이드는 생략해도 된다.
- 이미지를 생략하더라도 슬라이드에 있는 기술 내용은 본문에 반드시 포함한다.

## 슬라이드 내용 누락 방지 절차

각 Lecture/Exercise마다 페이지별 coverage table을 먼저 만든다.

```markdown
| 원본 | 페이지 | 문서 반영 위치 | 반영 상태 |
| --- | --- | --- | --- |
| Lecture 10 | p2 | `07_...md` / "아날로그 회로 개요" | 반영 |
| Lecture 10 | p3 | `07_...md` / "소신호 모델" | 반영 |
```

다음은 누락으로 간주한다.

- 슬라이드에 있는 수식이 본문에 없거나 의미 설명이 없는 경우
- 그래프의 축, 추세, 결론 중 하나라도 설명하지 않은 경우
- 표의 row/column 의미를 설명하지 않은 경우
- 회로도의 transistor 연결과 동작 조건을 설명하지 않은 경우
- 실습 슬라이드의 command/netlist가 빠진 경우
- 슬라이드가 "중요하지 않아 보인다"는 이유만으로 기술 내용을 생략한 경우

반대로 생략 가능한 것은 다음뿐이다.

- Moodle, 제출일, 강의실, 연락처
- 교수진 개인 약력
- 반복되는 표지/목차/closing slide
- 이미 본문에서 충분히 설명한 중복 그림

## 수식 설명 방식

수식은 반드시 다음 순서로 설명한다.

1. 수식을 정확히 다시 쓴다.
2. 각 항의 의미를 표로 정리한다.
3. 각 항이 커지거나 작아지면 결과가 어떻게 변하는지 설명한다.
4. 이 수식이 회로 설계에서 어떤 판단에 쓰이는지 말한다.
5. 슬라이드 식이 단순화된 식이면, 단순화된 이유와 한계를 밝힌다.

예시:

````markdown
```math
I_{ON} \sim \mu C_{ox}\frac{W}{L}(V_{DD}-V_{th})^2
```

| 항 | 의미 | 커지면 |
| --- | --- | --- |
| `mu` | carrier mobility | 전류가 증가한다 |
| `Cox` | gate oxide capacitance | gate control이 강해져 전류가 증가한다 |
| `W/L` | channel 폭/길이 비율 | 폭이 크고 길이가 짧을수록 전류가 증가한다 |
| `VDD - Vth` | overdrive voltage | 커질수록 전류가 크게 증가한다 |

이 식은 회로 delay를 줄이려면 `ION`을 키워야 한다는 점을 보여준다. 하지만 `Vth`를 낮추면 `ION`은 커지지만 `IOFF`도 커지므로, 성능과 leakage 사이 trade-off가 생긴다.
````

## 그래프 설명 방식

그래프는 단순히 "증가한다/감소한다"로 끝내지 않는다. 반드시 다음을 포함한다.

- x축과 y축이 무엇인지
- 각 곡선이 어떤 조건을 나타내는지
- 곡선의 주요 구간이 왜 그런 모양인지
- 그래프가 말하는 설계 결론이 무엇인지
- aging, temperature, voltage, load, duty cycle 중 어떤 물리 원인이 숨어 있는지

예시: bathtub curve라면 다음을 설명한다.

- 초기 고장률이 높은 이유: 제조 결함과 약한 소자의 조기 탈락
- 중간 구간이 낮고 평평한 이유: 정상 제품의 안정 동작
- 오른쪽에서 다시 상승하는 이유: BTI/HCI/TDDB/EM 같은 aging 누적
- aging이 빨라지면 wear-out 구간이 앞당겨지고 MTTF가 감소함

## 도표와 표 설명 방식

표는 내용을 그대로 옮기는 데서 끝내지 않는다.

- row와 column이 무엇을 비교하는지 설명한다.
- 표에서 가장 중요한 비교 축을 말한다.
- 시험 답안에서 어떻게 써먹을 수 있는지 한 문장으로 정리한다.

예시:

```markdown
| 구조 | Gate control | 장점 | 한계 |
| --- | --- | --- | --- |
| Planar | 약함 | 구조가 단순함 | leakage 증가 |
| FinFET | 강함 | scaling 가능 | fin 단위 이산 sizing |
| Nanosheet | 더 강함 | 더 좋은 electrostatics | 제조/레이아웃 제약 |
```

이 표의 핵심은 "구조가 발전할수록 gate control은 좋아지지만, layout과 제조 제약은 강해진다"는 점이다.

## 회로도 설명 방식

회로도는 다음 순서로 설명한다.

1. 입력, 출력, VDD, VSS/GND를 먼저 식별한다.
2. PMOS network와 NMOS network를 나누어 설명한다.
3. 어떤 입력 조합에서 어떤 transistor가 켜지는지 말한다.
4. output이 VDD로 pull-up되는지, VSS로 pull-down되는지 설명한다.
5. delay/power/aging 관점에서 어떤 transistor stack 또는 load가 중요한지 덧붙인다.

NAND 예시:

```markdown
PMOS 두 개는 병렬이고 NMOS 두 개는 직렬이다. 입력 둘 다 1일 때만 NMOS path가 완전히 켜져 output이 0이 된다. 하나라도 0이면 PMOS 중 하나가 켜져 output이 1이 된다. 그래서 이 회로는 NAND이다.
```

## 실습 스크립트와 netlist 원본 포함 방식

Exercise나 Assignment에 스크립트, netlist, shell command, Python 코드가 있으면 가능한 한 원본을 보존해서 문서에 포함한다. 단, 전체 파일이 너무 길면 핵심 block을 그대로 인용하고, 전체 파일 경로를 함께 적는다.

권장 형식:

````markdown
원본 파일: `Assignment/exercise3/projectfiles/nand_fresh_x2_x4_all.sp`

```spice
.subckt NAND_X2 A1 A2 VDD VSS ZN num_fins=1
m_p1 ZN    A1 VDD   VDD pmos_rvt nfin=num_fins
m_p2 ZN    A2 VDD   VDD pmos_rvt nfin=num_fins
m_n1 ZN    A1 INTER VSS nmos_rvt nfin=num_fins
m_n2 INTER A2 VSS   VSS nmos_rvt nfin=num_fins
.ends NAND_X2
```

이 block은 NAND standard cell의 transistor-level topology를 정의한다. PMOS는 병렬, NMOS는 직렬이므로 두 입력이 모두 1일 때만 output이 0으로 내려간다.
````

스크립트 설명 기준:

- command가 무엇을 실행하는지
- 입력 파일과 출력 파일이 무엇인지
- parameter sweep이 있다면 어떤 조건을 바꾸는지
- `.measure`, `.print`, `.dc`, `.tran`, `.mosra` 같은 지시어가 어떤 물리량을 측정하는지
- 결과 파일에서 어떤 column을 읽어야 하는지

중요: 원본 코드를 요약으로 대체하지 않는다. 학습에 필요한 핵심 줄은 반드시 code block으로 남긴다.

## 슬라이드 밖 이론 보강 방식

슬라이드 밖 이론 보강은 필요하지만, 과하게 확장하지 않는다. 보강은 슬라이드 이해에 직접 필요한 경우에만 한다.

보강이 필요한 경우:

- 슬라이드가 용어만 던지고 정의가 부족한 경우
- 수식의 항이 무엇인지 설명하지 않는 경우
- 그래프의 추세 원인이 생략된 경우
- 실습 command가 어떤 이론을 검증하는지 연결이 약한 경우
- 시험에서 물어볼 만한 인과관계가 슬라이드에 암시만 되어 있는 경우

보강 자료 우선순위:

1. 같은 Lecture/Exercise의 앞뒤 슬라이드
2. 이미 작성된 `Study Notes/` 문서
3. 사용자의 Assignment 결과와 report
4. 반도체/CMOS의 표준 교재 수준 지식
5. 외부 자료가 필요하면 공식 문서, 교재, 논문, 신뢰 가능한 reference 우선

외부 조사를 했다면 문서에 "슬라이드 밖 보강"임을 자연스럽게 드러낸다.

예:

```markdown
슬라이드에는 mobility가 낮아진다고만 적혀 있지만, 이유는 고온에서 silicon lattice vibration이 커져 carrier scattering이 증가하기 때문이다.
```

## 쉬운 설명 기준

설명은 고등학생 수준의 비전공자도 따라올 수 있어야 한다. 그러나 부정확한 비유로 대충 설명하면 안 된다.

해야 할 것:

- 한 문장에 한 개념만 담는다.
- 먼저 쉬운 정의를 주고, 그 다음 정확한 용어를 붙인다.
- 변수와 단위를 분명히 말한다.
- "왜 그런가"를 원인과 결과 순서로 설명한다.
- 추세는 "A가 커지면 B가 커진다/작아진다" 형태로 쓴다.
- 물리적인 이론에 근거를 두고 명쾌하게 설명한다. (관계식 언급)

피해야 할 것:

- 이상한 비유나 은유
- "느낌상", "대충", "마치 ..." 같은 흐린 설명
- 슬라이드 문장을 영어 그대로 번역만 하는 것
- 수식을 적고 해석 없이 넘어가는 것
- 그래프를 넣고 축/추세/의미 설명을 생략하는 것

좋은 문장 예:

```markdown
`Vth`가 낮아지면 channel을 만들기 위해 필요한 gate voltage가 줄어든다. 그래서 같은 `VDD`에서 `VDD - Vth`가 커지고, ON current가 증가한다. 하지만 OFF 상태에서도 carrier가 더 쉽게 흐르므로 leakage current도 증가한다.
```

나쁜 문장 예:

```markdown
Vth는 문 같은 것이고 낮으면 문이 잘 열린다.
```

비유 없이도 충분히 쉽게 설명할 수 있다.
